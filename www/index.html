For this project, our group was given a 3DR Hexacopter drone that will be programmed to fly to a fixed set of way stations and return to a base station using the program, Mission Planner. Upon reaching each way station the drone will take a series of images. These images are captured by implementing a Raspberry Pi that runs python scripts which communicate with a GPS module that activate the camera when a way station has been reached.

When the drone returns to the base station, the images that were captured and saved to the database are transmitted to a central location. A web interface will display the images to create a composite view of the area covered by the drone. Consider that when the drone is at a given position, you must be able to rotate the camera at least a few different directions so as to capture a wider swath of area, rather than a single shot in a fixed direction.
  1. Drone Navigation
    • The drone will have missions.
    • Each mission had a dateTime and a list of waypoints.
    • Each waypoint has the following information: latitude, longitude and altitude.
    • When leaving for a task, the drone must go to each waypoint, stabilize and take pictures.
    • The drone should get back to the start point after all points are visited.
  2. Camera Programming
    • When the drone gets in the right position and is stable, the camera must take a picture and save it locally.
  3. Web Server
    • Then the the drone gets back to the internet wifi range, the collected data must be upload to a web server. The data that will be uploaded is the images, time stamp and coordinates.
  1. Make drone fly.

  3. Attach camera to the pi and make a program to take pictures.
  4. Make crontab script to take picture n minutes.
  5. Setup up gps and make a program to display coordinates.

  7. Add Google maps api on website.

  9. Fly drone with pi, camera and gps attached.

Drone

The drone shown below is the Arducopter 3DR Hexa B, which is designed and manufactuered by the company 3D robotics. The drone ulitizes the APM 2.6 which is an open source autopilot system. With the help from Larry, we learned how to set up the drone to fly. (More information on this drone can be found here: http://store.3drobotics.com/products/arducopter-3dr-hexa-b-1

APM Mission Planner

The hexcopter has it's own gps module which is used to interface with Mission Planner. Mission Planner makes the hexcopter autonomous. Using this application, we can sent commands such as take off, waypoints, and land for a specific location on the map.
Because the gps is already used, we will need another gps that interface with the raspberry pi to collect the location of the drone to use with the camera.
  1. Camera Setup
    • Camera connection
    • Configuration
      • sudo raspi-config
      • Otpion 5 - Camera enable
    • The test camera command: raspistill -o image.jpg worked with no issues
    • Ref: http://www.raspberrypi.org/documentation/usage/camera/raspicam/raspistill.md
  2. GPS setup
    • GPS Connection using UART.The way I implement it was using the UART instead of a USB to TTL adapter cable because I had already used both of my USB ports.
    • Wiring
      • Connect Vin on GPS module to 3.3V or (5V) on the PI header. Connect both GND
      • Connect TX on the GPS to the RDX on Pi header.
      • Connect RX on the GPS to the TXD on PI header.
  3. Web Server
    • Then the the drone gets back to the internet wifi range, the collected data must be upload to a web server. The data that will be uploaded is the images, time stamp and coordinates.
